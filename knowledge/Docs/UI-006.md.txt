# Development Philosophy

We will **not** build screens.

We will build **capabilities**.

Each capability reaches production by passing through a maturity pipeline.

```text
Architecture

↓

Domain

↓

API

↓

Repositories

↓

State

↓

Components

↓

Screens

↓

Testing

↓

Production
```

No screen begins until its architectural foundations exist.

# What Comes Next

At this point, I **would not write React Native code yet**.

There is one architectural layer still missing that will dramatically accelerate implementation:

## The Blueprint Layer

Instead of immediately implementing components, define every screen as structured metadata.

For example:

```yaml
screen: booking-details

archetype: detail

capability: booking

layout:
  - Header
  - BookingSummary
  - Timeline
  - PaymentSummary
  - EvidenceGallery
  - ActionBar

permissions:
  - booking:view

queries:
  - GetBooking
  - GetTimeline

commands:
  - CancelBooking
  - CompleteBooking

events:
  - BookingUpdated

components:
  - BookingCard
  - Timeline
  - Badge
  - Section

supports:
  offline: true
  ai: true
  inspector: true
```

I would create one of these blueprint files for **every screen**.

Once that's done, we can automatically derive:

* Expo Router structure
* React Native screen scaffolds
* TanStack Query hooks
* Navigation registration
* Permission guards
* Telemetry events
* Storybook scenarios
* Test templates
* AI prompts for implementation

## My recommendation before writing any code

I think we've now completed roughly **80–85% of the architectural work**. The remaining high-value step is to introduce a **Blueprint Specification series (`BP-XXX`)**.

Instead of jumping into Expo, we would first create:

* **BP-001** — Screen Blueprint Specification (schema)
* **BP-002** — Component Blueprint Catalog
* **BP-003** — Screen Blueprints (one per screen from your design package)
* **BP-004** — Navigation Blueprint
* **BP-005** — Data Binding Blueprint (queries, commands, events per screen)

Those blueprints become the deterministic input for implementation. Once they exist, generating the React Native application becomes a largely mechanical process instead of an architectural exercise.
