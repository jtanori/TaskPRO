# Architectural Principle

One of the most important decisions we can make now is to treat **every user action as an intent that produces a domain event**, rather than a direct database mutation.

The flow becomes:

```
User Intent
        │
        ▼
Application Use Case
        │
        ▼
Domain Validation
        │
        ▼
Domain Event
        │
        ▼
Persistence
        │
        ▼
Realtime Update
        │
        ▼
UI Projection
```

This aligns naturally with the event-driven philosophy we've been using in SYNTH and gives us a clean separation between business logic, persistence, and presentation.
