# UI-001 — Application Shell & Navigation Architecture v0.1

## Purpose

This document defines the **structural skeleton** of the application.

It does **not** define colors, buttons, typography, or spacing.

Instead, it defines:

* Navigation
* Shell
* Screen hierarchy
* Layout zones
* Global UI services
* Platform behaviors

Think of this as the operating system of TaskPRO.

The Design System (DS-001+) provides the visual language, while UI-001 defines how the application is assembled.

---

# Design Philosophy

The application should feel like an **AI-native workspace**, not a collection of mobile screens.

Instead of navigating between isolated pages, the user moves through **contexts**.

A context owns:

* navigation
* state
* actions
* AI assistant
* overlays
* workflows

---

```text
TaskPRO App
```

Every screen lives inside the shell.

# Architectural Observation

This is where I would diverge slightly from a conventional Expo application.

Rather than centering the shell around **navigation**, I would center it around **contexts**.

A navigation stack answers *"Where am I?"*

A context answers:

* What capability am I in?
* What data is active?
* What actions are available?
* What should the AI assistant know?
* What should notifications, overlays, and realtime subscriptions be scoped to?

That subtle distinction is what makes the application feel AI-native rather than simply mobile-native. It also aligns with the architectural direction we've established throughout the previous documents.
