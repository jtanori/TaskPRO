# TASKPRO-SCREEN-001 — TaskPRO Screen Specification Catalog v0.1

## Purpose

The Screen Specification Catalog defines the complete TaskPRO application screen system.

This artifact translates:

* design files
* React Native implementation patterns
* component catalog
* navigation structure
* user workflows

into a reusable screen-level specification.

The goal is not pixel documentation. The goal is to establish the **screen architecture** required for:

* engineering implementation
* AI-assisted generation
* UX validation
* feature expansion
* design/code synchronization

---

# 1. Screen System Overview

## 1.1 Screen Architecture Principles

TaskPRO screens follow the structure:

```text
Screen Context

        ↓

Information Hierarchy

        ↓

Primary User Goal

        ↓

Primary Action

        ↓

Supporting Actions
```

Every screen should answer:

1. Where am I?
2. What information matters?
3. What can I do next?

---

# 1.2 Screen Categories

TaskPRO screens are organized into:

```text
1. Entry Screens
2. Authentication Screens
3. Home / Dashboard Screens
4. Discovery Screens
5. Service Screens
6. Marketplace Screens
7. Workflow Screens
8. Communication Screens
9. Profile Screens
10. Settings Screens
```

---

# TASKPRO-FLOW-001 — TaskPRO User Workflow Specification v0.1

## Purpose

The Workflow Specification defines how users move through TaskPRO to accomplish meaningful goals.

While:

* **DS-001 → DS-005** define the design language
* **COMP-001** defines reusable UI building blocks
* **SCREEN-001** defines individual screens

this artifact defines the **behavioral system connecting screens together**.

The workflow layer answers:

* What is the user trying to accomplish?
* What screens are involved?
* What decisions occur?
* What states exist?
* What happens when something succeeds, fails, or requires intervention?

---

# 1. Workflow System Overview

## 1.1 Workflow Principles

TaskPRO workflows follow:

```text
Intent

↓

Discovery

↓

Decision

↓

Action

↓

Confirmation

↓

Progress

↓

Completion
```

---

## 1.2 Workflow Design Rules

### Minimize User Uncertainty

At every step users should understand:

* current status
* next action
* expected outcome

---

### Preserve Progress

Users should not lose completed work.

Examples:

* partially completed forms
* selected services
* uploaded information

---

### Make States Visible

Every workflow has explicit states.

Example:

```text
Requested

↓

Accepted

↓

Scheduled

↓

In Progress

↓

Completed
```

---

### Recovery Is Part of the Flow

Failures are expected.

Every workflow defines:

* failure states
* recovery actions
* retry paths

---

# 2. Workflow Model

Every TaskPRO workflow consists of:

```text
Workflow

├── Entry Point
├── User Intent
├── Screens
├── Actions
├── Decisions
├── States
├── Events
└── Completion Criteria
```
