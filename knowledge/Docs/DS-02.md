# TASKPRO-DS-002 — TaskPRO Component Specifications v1.0

**Status:** Approved Architecture
**Owner:** Design System
**Audience:** Product Design, React Native, Frontend, AI Code Generation, QA

---

# 1. Purpose

This document defines every reusable UI component within TaskPRO.

A component is a reusable, composable building block derived from the Design Token System (TASKPRO-DS-001).

Every screen in TaskPRO must be composed exclusively from components defined in this specification.

No screen should introduce one-off visual patterns.

---

All components follow the same hierarchy:

# 39. Implementation Rules

✅ Components must consume only Design Tokens.

✅ No inline styling outside exceptional cases.

✅ Every reusable UI element belongs in the component library.

✅ Feature teams extend through composition, never duplication.

✅ Business logic remains outside presentation components.

✅ Components must be deterministic and independently testable.

---

# 40. Component Inventory Summary

The TaskPRO Design System comprises five primary layers:

* **Foundation:** layout, spacing, typography, surfaces, containers.
* **Primitive Components:** buttons, inputs, icons, avatars, lists, cards.
* **Composite Components:** search bars, navigation bars, bottom sheets, dialogs, progress indicators.
* **Domain Components:** marketplace, booking, technician, messaging, AI, payments, dashboards, profile.
* **Feature Assemblies:** screen-specific compositions built exclusively from lower-level components.

This layered architecture ensures consistency, scalability, and AI-assisted code generation while preventing duplication and maintaining a single source of truth for the entire application.
